# 会话功能配置

v1.2.3 版本新增了会话功能的全局配置能力，允许开发者通过后端配置完全启用或禁用多会话功能。

## 功能说明

通过 `enableChatSession` 配置项，可以控制整个多会话功能在UI层面的显示与隐藏。当设置为 `false` 时，组件头部的"历史会话"和"新建聊天"图标将不会显示，用户无法进行多会话操作。

## 配置方式

此配置项由后端通过 `getAgentInfo` 接口返回，位于 `IAgentInfo` 对象的 `conversationSettings` 属性中。

### 配置结构

```json
{
  "conversationSettings": {
    "enableChatSession": true, // 或 false
    // ... 其他会话相关设置
  }
  // ... 其他配置项
}
```

### 配置说明

- `enableChatSession`: `boolean` 类型
  - `true` (默认值): 启用多会话功能，显示会话管理相关UI元素
  - `false`: 禁用多会话功能，隐藏会话管理相关UI元素

## 使用场景

此功能适用于以下场景：

1. **单次问答模式**: 对于只需要简单问答交互的场景，可以禁用复杂的多会话功能，提供更简洁的用户界面。
2. **权限控制**: 根据用户角色或权限动态控制是否允许使用多会话功能。
3. **功能灰度发布**: 在新功能上线初期，可以针对部分用户或环境启用多会话功能进行灰度测试。

## UI影响

当 `enableChatSession` 设置为 `false` 时，以下UI元素将被隐藏：

- 头部栏中的"历史会话"图标按钮
- 头部栏中的"新建聊天"图标按钮
- 相关的会话管理交互功能将不可用

## 注意事项

1. 此配置项仅控制UI显示，不影响组件内部的会话管理逻辑。
2. 配置项需要通过后端接口返回，无法通过前端props直接设置。
3. 如需动态切换此配置，需要刷新页面或重新初始化组件以获取最新的配置信息。
