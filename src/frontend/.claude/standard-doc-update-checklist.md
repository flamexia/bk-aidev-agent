# 标准文档更新检查清单

此清单用于确保每次功能开发后的文档更新都遵循完整流程，保持代码与文档的一致性。

## 第一步：变更分析
- [ ] 确定变更类型：新功能 / 功能增强 / API 变更 / Bug 修复
- [ ] 评估变更对现有文档的影响范围
- [ ] 确定需要更新的文档类型：API 文档 / 指南文档 / 示例代码 / 类型定义

## 第二步：API 文档更新
- [ ] 检查并更新 `web/docs/api/props.md` - 添加/修改组件属性
- [ ] 检查并更新 `web/docs/api/methods.md` - 添加/修改组件方法
- [ ] 检查并更新 `web/docs/api/events.md` - 添加/修改组件事件
- [ ] 检查并更新 `web/docs/api/types.md` - 添加/修改类型定义
- [ ] 确保每个 API 条目都有清晰的描述、类型、默认值和版本号标记
- [ ] 检查 `ai-blueking/src/types/index.ts` - 确认类型定义与文档一致

## 第三步：指南文档更新
- [ ] 检查 `web/docs/guide/` 目录下的相关指南
- [ ] 更新现有指南或创建新指南页面
- [ ] 确保示例代码可运行且与 API 保持一致
- [ ] 检查并更新导航和侧边栏配置

## 第四步：版本日志更新
- [ ] 更新 `ai-blueking/CHANGELOG.md` - 添加版本更新日志
- [ ] 更新 `web/docs/changelog.md` - 添加版本更新日志
- [ ] 确保类型定义中的版本标记（如 `@since v1.2.9`）与 changelog 一致

## 第五步：验证与链接修复
- [ ] 全局搜索旧文档链接，修复所有死链
- [ ] 检查 API 文档中的交叉引用
- [ ] 运行本地文档构建验证 (`npm run docs:dev` 或类似命令)
- [ ] 确保所有示例代码正常工作

## 第六步：最终检查
- [ ] 确认所有相关文件都已提交
- [ ] 检查文档格式和风格一致性
- [ ] 验证版本号和发布日期正确
